# 🚀 ربات ردیاب پروازهای نظامی با هوش مصنوعی Gemini

<div dir="rtl">

## 📋 معرفی

این ربات یک سیستم هوشمند تحلیل پروازهای نظامی است که با استفاده از **هوش مصنوعی Google Gemini 2.0** و **API های زنده ردیابی پرواز**، پروازهای نظامی را به صورت real-time شناسایی، تحلیل و گزارش می‌کند.

### ✨ ویژگی‌های اصلی

- 🤖 **تحلیل هوشمند با Gemini AI** - شناسایی دقیق نقش، مدل و اهمیت استراتژیک هواپیماها
- 🔄 **چند کلید API با سوییچ خودکار** - در صورت محدود شدن یک کلید، به طور خودکار به کلید بعدی می‌رود
- 💾 **سیستم کش هوشمند** - ذخیره تحلیل‌های AI برای استفاده مجدد و کاهش هزینه
- 🛡️ **Fallback به دیتابیس محلی** - در صورت عدم دسترسی به AI، از دیتابیس محلی استفاده می‌کند
- 📊 **گزارش‌دهی پیشرفته** - گزارش‌های ساعتی و روزانه با آمار کامل
- 🎯 **فیلترینگ هوشمند** - حذف پروازهای داخلی آمریکا و فیلتر بر اساس اهمیت استراتژیک
- 🗺️ **تحلیل موقعیت جغرافیایی** - تشخیص مناطق استراتژیک و ارائه زمینه عملیاتی
- 📈 **پایگاه داده پیشرفته SQLite** - ذخیره تاریخچه کامل پروازها و تحلیل‌ها

## 🔧 نصب و راه‌اندازی

### پیش‌نیازها

- **Python 3.8+** (توصیه می‌شود: Python 3.10+)
- **حساب کاربری Telegram** و **Bot Token**
- **یک یا چند کلید API از Google Gemini** ([دریافت کلید](https://aistudio.google.com/app/apikey))

### مراحل نصب

#### 1. کلون کردن یا دانلود پروژه

```bash
cd /workspace
# فایل‌ها از قبل آماده هستند
```

#### 2. نصب کتابخانه‌های مورد نیاز

```bash
pip install -r requirements.txt
```

#### 3. پیکربندی ربات

فایل `config.py` را باز کرده و تنظیمات زیر را انجام دهید:

```python
# توکن ربات تلگرام
TELEGRAM_BOT_TOKEN = "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz"

# آیدی کانال تلگرام (با @ یا ID عددی)
TELEGRAM_CHANNEL_ID = "@military_flights"  # یا "-1001234567890"

# آیدی ادمین برای دریافت گزارش‌های روزانه
ADMIN_USER_ID = 7717672777

# کلیدهای API جمنای (می‌توانید چندین کلید اضافه کنید)
GEMINI_API_KEYS = [
    "AIzaSy.....................",  # کلید اول
    "AIzaSy.....................",  # کلید دوم
    "AIzaSy.....................",  # کلید سوم
]
```

#### 4. اجرای ربات

```bash
python bot.py
```

اگر همه چیز درست پیکربندی شده باشد، خروجی زیر را خواهید دید:

```
✅ Configuration validated successfully!
🚀 Military Flight Tracker Bot v15.0 (Gemini AI Edition)
✅ Database initialized: military_flights.db
✅ Gemini AI Manager initialized with 3 API key(s)
🎯 Bot is ready! Press Ctrl+C to stop.
```

## 🎯 نحوه کار سیستم

### جریان کاری (Workflow)

```
1. دریافت داده از ADSB.lol API
         ↓
2. فیلتر کردن (حذف USA، تکراری‌ها)
         ↓
3. بررسی کش: آیا این هواپیما قبلاً تحلیل شده؟
         ↓
4. اگر نه → ارسال به Gemini AI
         ↓
5. دریافت تحلیل هوشمند (نقش، مدل، اهمیت)
         ↓
6. بررسی اهمیت استراتژیک (حداقل 5/10)
         ↓
7. ارسال اعلان به کانال تلگرام
         ↓
8. ذخیره در دیتابیس برای استفاده آینده
```

### سیستم چند کلید API

ربات به صورت هوشمند بین کلیدهای API سوییچ می‌کند:

- اگر کلید فعلی محدود شد (Quota) → **سوییچ خودکار به کلید بعدی**
- اگر خطا داد → **بعد از 3 تلاش، سوییچ می‌کند**
- اگر همه کلیدها تمام شد → **Fallback به دیتابیس محلی**

## 📊 گزارش‌های خودکار

### گزارش ساعتی (Hourly Report)

هر ساعت یکبار به کانال ارسال می‌شود و شامل:
- تعداد بررسی‌ها
- پروازهای شناسایی شده
- آمار AI (موفقیت، کش)
- وضعیت کلید API فعال

### گزارش روزانه (Daily Report)

هر روز راس ساعت 00:00 UTC به **ادمین** ارسال می‌شود:
- لیست هواپیماهایی که شناسایی کامل نشدند
- فرمت فایل `.txt` قابل دانلود
- پاکسازی خودکار رکوردهای قدیمی

## 🗄️ ساختار پایگاه داده

ربات از **5 جدول اصلی** استفاده می‌کند:

### 1. `spotted_aircraft`
ذخیره هواپیماهای رصد شده برای جلوگیری از تکرار اعلان

### 2. `ai_analysis_cache` ⭐
**مهم‌ترین جدول**: ذخیره تحلیل‌های AI برای استفاده مجدد

```sql
- persian_role          (نقش به فارسی)
- aircraft_model        (مدل کامل)
- operator_analysis     (اپراتور)
- strategic_importance  (اهمیت 1-10)
- times_used            (تعداد استفاده)
```

### 3. `flight_history`
تاریخچه کامل تمام پروازها (موقعیت، ارتفاع، سرعت)

### 4. `unidentified_aircraft`
هواپیماهایی که نتوانستند شناسایی شوند

### 5. `statistics`
آمار عملکرد ربات (ساعتی)

## ⚙️ تنظیمات پیشرفته

### تنظیم فاصله بررسی

```python
# در config.py
POLL_INTERVAL_SECONDS = 120  # هر 2 دقیقه (پیشنهادی)
```

### تنظیم حداقل اهمیت برای اعلان

```python
MIN_STRATEGIC_IMPORTANCE = 5  # فقط پروازهای مهم (5-10)
```

### فعال/غیرفعال کردن AI

```python
ENABLE_AI_ANALYSIS = True   # True = استفاده از Gemini
USE_AI_CACHE = True         # True = استفاده از کش
```

### تنظیم مدل Gemini

```python
GEMINI_MODEL = "gemini-2.0-flash-exp"  # سریع‌ترین
# یا
GEMINI_MODEL = "gemini-2.0-pro-exp"    # دقیق‌ترین
```

## 🔍 عیب‌یابی (Troubleshooting)

### خطا: "Configuration validation failed"

**راه‌حل**: فایل `config.py` را باز کرده و مطمئن شوید:
- `TELEGRAM_BOT_TOKEN` را با توکن واقعی جایگزین کرده‌اید
- `TELEGRAM_CHANNEL_ID` را تنظیم کرده‌اید
- حداقل یک `GEMINI_API_KEY` معتبر وارد کرده‌اید

### خطا: "No valid Gemini API keys configured"

**راه‌حل**: حداقل یک کلید API معتبر از [Google AI Studio](https://aistudio.google.com/app/apikey) دریافت کنید.

### ربات پروازی شناسایی نمی‌کند

**علل محتمل:**
1. **API ادغام شده محدود است** → سوییچ خودکار روی کلید بعدی
2. **فیلتر اهمیت استراتژیک** → `MIN_STRATEGIC_IMPORTANCE` را کاهش دهید
3. **فیلتر USA** → پروازهای داخل آمریکا نمایش داده نمی‌شوند

### لاگ‌ها کجا هستند؟

فایل `military_tracker.log` در همان دایرکتوری ربات ساخته می‌شود.

## 📈 بهینه‌سازی و توصیه‌ها

### برای کاهش هزینه API

1. **فعال کردن کش**: `USE_AI_CACHE = True`
2. **افزایش فاصله بررسی**: `POLL_INTERVAL_SECONDS = 180` (3 دقیقه)
3. **افزایش حداقل اهمیت**: `MIN_STRATEGIC_IMPORTANCE = 6`

### برای دقت بیشتر

1. **استفاده از مدل Pro**: `GEMINI_MODEL = "gemini-2.0-pro-exp"`
2. **کاهش Temperature**: `GEMINI_TEMPERATURE = 0.2`

### برای سرعت بیشتر

1. **استفاده از Flash**: `GEMINI_MODEL = "gemini-2.0-flash-exp"`
2. **کاهش Timeout**: `GEMINI_TIMEOUT = 10`
